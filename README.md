# Tree in a row

Консольная игра "Три в ряд" на Haskell.

Игрок обменивает местами соседние элементы на поле `3x3`.  
Когда в строке собирается тройка одинаковых чисел, она удаляется, пустые ячейки заполняются новыми случайными значениями, а счёт увеличивается.

## Возможности

- Поле `3x3` с числами от `1` до `3`.
- Обмен только соседних элементов (по горизонтали или вертикали).
- Автоматический поиск горизонтальных троек.
- Удаление троек и генерация новых элементов.
- Подсчёт очков по количеству найденных троек.
- Выход из игры по команде `q`.

## Требования

- [GHC](https://www.haskell.org/ghc/) (проект собран с `ghc-9.14.1`)
- [Cabal](https://www.haskell.org/cabal/)

## Установка зависимостей

В проекте используется библиотека `random`, она уже указана в `tree-in-a-row.cabal`.  
Обычно достаточно выполнить сборку, и Cabal сам подтянет зависимости.

## Запуск

Из корня проекта:

```bash
cabal run tree-in-a-row
```

Либо:

```bash
cabal build
cabal exec tree-in-a-row
```

## Правила ввода

Во время хода введите координаты двух соседних клеток в формате:

```text
строка1 столбец1 строка2 столбец2
```

Пример:

```text
1 1 1 2
```

Это обменяет элементы в первой строке между первым и вторым столбцом.

Важно:

- Координаты вводятся в диапазоне от `1` до `3`.
- Элементы должны быть соседними.
- Для выхода введите `q`.

## Пример игрового цикла

1. Печатается текущее поле.
2. Игрок вводит координаты обмена.
3. Если обмен корректен, поле обновляется.
4. Если появились тройки, они удаляются, поле дозаполняется.
5. Счёт увеличивается, цикл повторяется.

## Ограничения текущей версии

- Поиск троек реализован только по горизонтали.
- Поле фиксированного размера `3x3`.
- Нет проверки на некорректный числовой ввод (например, буквы вместо чисел).
